# Template: RCA Effort Scale 0-100

> Каноничная шкала измерения **усилия владельца / человека** в любом разрыве между ожиданием и реальностью. Используется в `Owner effort digest` (mandatory delivery section #10) и в `Gap table → Next action`.

## Принцип

Усилие владельца — это не «время в минутах», а **когнитивная + механическая нагрузка** на конкретного человека в конкретной роли.

**Главное правило:** любой клик / открытие документа / переключение контекста владельца = **минимум 50 единиц**. Ноль — только когда агент / процесс делает всё сам без прерывания владельца.

## Шкала

| Балл | Что значит | Примеры |
|---|---|---|
| **0** | Полная автоматизация — владелец не прерывается | Агент сам читает чат, делает запись в .beads, отчитывается в digest |
| **5** | Только информация в digest, действие не нужно | «Готово, см. ссылку — никаких действий не требуется» |
| **10** | Одно подтверждение / один клик «approve» | Click → done |
| **20** | 1 решение из 2-3 готовых вариантов | «Выбери A или B» |
| **30** | 2-3 клика / форма с короткими полями | Заполнить ticket с 3 полями |
| **50** | **Любой клик владельца минимум** — открыть документ, переключить контекст, прочитать > 1 экрана | Открыть PR + прочитать diff, перейти в другой инструмент |
| **70** | Ручное расследование — нужно собирать факты, читать переписку, проверять гипотезы | Разобраться почему упал релиз |
| **90** | Переделка — текущая работа невалидна, надо начать заново | «Этот документ не работает, перепиши» |
| **100** | Полная переделка с нуля + восстановление контекста | «Удали всё и начни с другой стороны» |
| **+50** | Надбавка за легенду / отдельный мини-словарь | Голые коды без расшифровки в той же строке |
| **+100** | Провал доверия — нужно восстанавливать отношения | «Я тебе больше не верю, давай сначала» |

## Calibration tests

Если не уверен какой балл поставить — пройди эти тесты:

1. **Опрос «сколько секунд?»** — если отвечаешь «секунды» = 0-10. Если «минуты» = 30-50. Если «часы» = 70-90. «День или больше» = 100+.
2. **Опрос «сколько контекстов переключить?»** — 0 = осталось в текущем. 1 переключение = +20. 2 = +40. 3+ = +60.
3. **Опрос «надо ли восстанавливать какие-то факты?»** — нет = +0. Из памяти = +10. Из переписки = +30. Из логов / data = +50. Из нескольких источников + синтез = +70.
4. **Опрос «есть ли риск ошибиться без помощи?»** — нет = +0. Есть, обратимо = +10. Есть, необратимо = +50.

Сумма = балл (округление до ближайшего из шкалы).

## Применение в Owner effort digest

Финальный блок delivery всегда содержит таблицу:

| Next action | Усилие человека (0-100) | Что агент может сделать сам |
|---|---|---|
| Подтвердить лицензию (option A или B) | 20 (выбор из вариантов) | Я предложил вариант + объяснил → теперь только yes/no |
| Прочитать README перед commit | 50 (открытие + чтение) | Я могу commit без read, но риск что не понравится |
| Обновить версию в CITATION.cff | 0 | Сам обновлю при release |
| Перечитать standards/01-gap-theory.md | 70 (расследование) | Я не могу избавить — это часть твоей работы как автора |

Если все строки = 0 → блок звучит как **«Нет шагов для человека»** + причина (нет хвоста / только информация).

## Anti-patterns

- ❌ Поставить 0 на «прочитай PR на 1500 строк» (это 70-90)
- ❌ Поставить 50 на «нажми Enter» (это 10)
- ❌ Не обозначить надбавки (+50 за легенду / +100 за переделку) — занижает реальную нагрузку
- ❌ Использовать аббревиатуру `OEU` (operator effort units) — заголовок колонки **«усилие человека (0-100)»**, читаемо
