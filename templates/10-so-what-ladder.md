# Template: So-What Outcome Ladder

> Метод отсечения псевдо-outcomes от настоящих outcomes. Для каждого заявленного outcome задаём «И что с того?» 5 раз. Если ответ скатывается обратно к промежуточному шагу (output / артефакт / процесс) — это не outcome.

## Принцип

| Уровень | Что значит | Пример |
|---|---|---|
| **Output** | Артефакт / событие / документ — то что существует наблюдаемо | «Создан README в репо» |
| **So what 1** | Что это меняет на 1 уровень глубже | «Контрибьюторы понимают как использовать» |
| **So what 2** | Что это меняет на 2 уровень глубже | «Появляются первые PR от внешних контрибьюторов» |
| **So what 3** | Что это меняет на 3 уровень глубже | «Сообщество расширяется и применяет метод» |
| **So what 4** | Что это меняет на 4 уровень глубже | «Метод цитируется в курсах / книгах / B2B консалтинге» |
| **So what 5** | Real outcome — что меняется в моей жизни / роли | «Я становлюсь признанным authority в gap theory + это даёт мне доход и repo of record» |

5 уровней — потому что меньше = поверхностно, больше = уже спекулятивно.

## Шаблон

### Заявленный output / outcome
```
[То что я считаю outcome своей работы]
Например: «опубликовать README»
```

### Ladder

```
Output: [output]
       └─ So what 1? → [уровень 1]
            └─ So what 2? → [уровень 2]
                 └─ So what 3? → [уровень 3]
                      └─ So what 4? → [уровень 4]
                           └─ So what 5? → [real outcome для моей роли]
```

### Самопроверка

| Тест | Прошёл? |
|---|---|
| Уровень 5 описывает изменение в моей роли / жизни / бизнесе, а не артефакт | yes / no |
| Уровень 5 измерим (не абстрактное «улучшение») | yes / no |
| Каждый шаг логично следует из предыдущего (нет «магических скачков») | yes / no |
| Уровень 5 ≠ уровню 1-4 (нет циклов) | yes / no |
| Если уровень 5 не материализуется — я бы не делал output | yes / no |

Все 5 yes = настоящий outcome. Хоть один no = это псевдо-outcome или output.

## Worked examples

### Хороший пример (real outcome)

```
Output: «опубликовать gap theory в OSS-репо»
└─ So what 1? → внешние команды могут применять метод сами без меня
   └─ So what 2? → метод фиксируется как mine с сохранением атрибуции
      └─ So what 3? → создаётся repo of record на который ссылаются курсы и продукты
         └─ So what 4? → привлекает контрибьюторов и кейсы расширяющие банк примеров
            └─ So what 5? → усиливает позиционирование Gap Theory как named methodology — основа для книги/курса/B2B-продукта (real outcome для моей роли author + advisor)
```

✅ Уровень 5 — конкретное изменение в позиционировании автора
✅ Каждый шаг логичен
✅ Если real outcome не материализуется — публикация теряет смысл
**Verdict:** real outcome, можно делать output.

### Плохой пример (псевдо-outcome)

```
Output: «провести встречу команды»
└─ So what 1? → команда обсудит проект
   └─ So what 2? → у всех будет понимание
      └─ So what 3? → команда выровнена
         └─ So what 4? → проект идёт лучше
            └─ So what 5? → проект успешен
```

❌ Уровень 5 = «успешен» — абстрактно, не измеримо
❌ Уровни 1-3 циклически связаны (обсудят → понимание → выровнены — синонимы)
❌ Шаги 4-5 — магические скачки (как именно «лучше» становится «успехом»?)

**Verdict:** псевдо-outcome. Встреча — это output. Real outcome не сформулирован.

**Fix:** заменить на:
```
Output: «провести встречу команды»
└─ So what 1? → команда увидит конкретную метрику X упала на 12%
   └─ So what 2? → команда выберет 1 из 3 предложенных причин
      └─ So what 3? → owner назначен на root cause investigation
         └─ So what 4? → за 7 дней появится design injection в стандарт
            └─ So what 5? → подобный инцидент не повторится — RCA закрыта (real outcome для PM роли)
```

## Anti-patterns

- ❌ So what 5 = «больше денег» без объяснения механики
- ❌ So what 1-5 — синонимы (обсудим / поймём / разберёмся / выровняемся / успешны)
- ❌ Магические скачки между уровнями
- ❌ Уровень 5 — артефакт («появится дашборд») вместо изменения роли
- ❌ Длина ladder = 2-3 — недостаточно глубины

## Outcome Ladder Gate (mandatory)

При финальной доставке (mandatory section #9 «Hypothesis falsification»), при `verdict: confirmed` или `partially confirmed` — обязательно применить so-what ladder для подтверждения что output превращается в real outcome.
