# Root Cause First — устраняй корневую причину, не срезай углы

## Назначение

**JTBD (Jobs To Be Done — задача, которую решает владелец):** Когда агент сталкивается с проблемой (ошибка билда, несовместимость, падение теста, сломанный процесс), он должен найти и устранить **корневую причину**, а не ограничиваться обходным решением. Workaround создаёт технический долг, скрывает реальную проблему и ломается при следующем изменении.

**Корневая проблема, которую решает скилл:** Агенты склонны к path of least resistance — при первом отказе переключаются на workaround, теряя из виду корневую причину. Владелец вынужден повторно разбираться с той же проблемой в следующей сессии.

## When to Use

**Ключевые слова-триггеры (любое из):**
- `корневая`, `корневую`, `корневая причина`
- `устраняй корневую`, `почини по-настоящему`
- `не срезай углы`, `не обходи проблему`
- `root cause`, `fix the root`, `don't cut corners`
- `don't workaround`, `не костыль`
- `два решения` (= локальное + корневое)
- владелец недоволен обходным решением

**Автоматический триггер (без явного запроса):**
- Агент собирается заменить один инструмент другим из-за ошибки (например, swc → babel из-за несовместимости Node)
- Агент планирует `|| true`, `try/catch с ignore`, `skip`, `disable` вместо исправления
- Агент предлагает «пока используем X, потом починим»

## Обязательный протокол

### Шаг 1 — Диагностика корневой причины

Перед любым workaround агент ОБЯЗАН:

1. **Назвать конкретную ошибку** — полный текст, не пересказ
2. **Задать 5 Why** — минимум 3 уровня «почему»:
   - Почему ошибка возникла?
   - Почему предыдущий уровень стал возможен?
   - Почему это не было предотвращено раньше?
3. **Записать корневую причину** — одно предложение, проверяемое

**Пример (из реального инцидента 2026-04-10):**
```
Ошибка: dlopen swc.darwin-arm64.node — incompatible architecture (have 'arm64', need 'x86_64')
Why 1: Node пытается загрузить arm64 binary, но сам запущен как x86_64
Why 2: Volta скачала x86_64 Node, потому что сама Volta установлена под Rosetta
Why 3: Volta installer не детектирует нативную архитектуру на Apple Silicon
Корневая причина: Volta x86_64 → Node x86_64 → конфликт с arm64 native modules
```

### Шаг 2 — Два решения параллельно

Агент ОБЯЗАН предложить **оба** решения:

| Тип | Что | Когда применять |
|---|---|---|
| **Быстрое (локальное)** | Workaround, который разблокирует работу прямо сейчас | Немедленно |
| **Корневое** | Устранение настоящей причины | Сразу после локального или параллельно |

**Порядок:**
1. Применить быстрое решение → разблокировать владельца
2. **Не останавливаться** → сразу приступить к корневому
3. Проверить корневое решение → подтвердить что workaround больше не нужен

### Шаг 3 — Проверка устранения

После корневого решения агент ОБЯЗАН:
- Убрать workaround (если он больше не нужен)
- Проверить, что проблема решена без workaround
- Записать в Reasoning Log: `Root cause eliminated: {что именно}`

---

## Hard fail conditions

1. **Workaround без диагностики** — агент применил обходное решение без 5 Why и без попытки корневого
2. **«Потом починим»** — агент остановился на workaround и перешёл к другой задаче
3. **Замена вместо починки** — агент заменил библиотеку/инструмент вместо решения несовместимости
4. **`|| true` / `try {} catch {}` / `--force` / `--skip`** — подавление ошибки без диагностики
5. **Один уровень Why** — агент назвал симптом, не добрался до причины

---

## Допустимые исключения

Workaround **без** корневого решения допустим **только** если:
- Корневая причина в upstream (чужой пакет, чужой сервис) И агент явно это зафиксировал
- Владелец явно сказал «сейчас не чини, только workaround»
- Исправление корневой причины требует изменений в другом репозитории/системе

В этих случаях агент ОБЯЗАН:
- Записать корневую причину в `.beads` или `ai.incidents.md`
- Добавить TODO с пометкой `root-cause-pending`
- Уведомить владельца: «Корневая причина: {X}. Workaround: {Y}. Когда upstream починит: {условие}»

| `if platform != 'darwin': skip` | Пропустил тест на macOS | Починить platform-specific код |

## Reasoning Log Protocol (обязательно)

При каждом исполнении этого скилла:

```markdown
### Root Cause Analysis — [дата UTC]
| # | Симптом | 5 Why (минимум 3 уровня) | Корневая причина | Быстрое решение | Корневое решение | Статус |
|---|---------|--------------------------|-------------------|-----------------|------------------|--------|
```
